Задание на дизайн — PetHealthAI (редизайн)
Полная версия. Заменяет прежнее «Задание на дизайн». Визуальный язык — glassmorphism + мятный акцент; технические рельсы проекта неизменны. Стек: Expo SDK 54 / RN 0.81 / React 19 / React Navigation 7 / Supabase / AI через OpenRouter. Темы — light + dark через theme-контекст. Рельсы: функциональные компоненты + хуки; экраны в AppNavigator (AI — в AssistantStack); данные через usePets/PetContext и useUserProfile/userService; геймификация — useBadges/useLoyaltyPoints/useCharity; все строки — i18next (en/ru); AI — OpenRouter (free → paid). Подробные токены — в PetHealthAI_visual_foundation.md (этот документ их использует).


0. Принципы
Тёплый wellness-тон, мягкое стекло на декоративных/оверлейных поверхностях.
Данные — на сплошных контрастных карточках, не на стекле (графики, цифры, медзаписи, статусы, формы).
Один бренд-акцент (мятный), консистентно. Семантика здоровья (ok/warn/danger) отдельно от акцента.
Тактильные 3D-иллюстрации — только в наградах/charity.
Доступность поверх эстетики: контраст ≥ WCAG AA в обеих темах.

2. Глобальные паттерны
2.1 Навигация / таб-бар
Нижний таб-бар, 5 вкладок: Главная · Здоровье · Ассистент · Активность · Профиль.
Поверхность — glass (BlurView), активная иконка — accent, неактивная — t3.
Регистрация стеков в AppNavigator; AI-экраны — в AssistantStack.
2.2 Компоненты-примитивы
GlassCard, Card (solid), IconChip (круглый accent-tint + accent-иконка), Button (primary/secondary/ghost, pill), Badge/StatusBadge, Input/Select/DateTimePicker, Chip (фильтры), Toggle, ProgressBar, Skeleton, BottomSheet, Toast, AutocompleteInput. Состояния каждого: default/press/disabled/loading/empty/error.
2.3 Состояния (единые)
Loading — Skeleton в форме контента (карточки/строки/график).
Empty — иллюстрация + текст + подсказка действия.
Error — иконка + текст + «Повторить».
Haptics — успех (двойной), переключение (лёгкий) на ключевых действиях.
2.4 Темы
Обе равноправны. useTheme(); стекло в dark затемнённое; контраст t1/t2 проверяется на surface и на стекле.


3. Онбординг и вход (кросс-секционное)
3.1 Вход / регистрация
Логин/регистрация; фон — мятный градиент, карточка формы — glass.
Поля — Input (pill), CTA — Button primary.
Состояния: валидация, error, loading.
3.2 QR-онбординг (B2B, ядро привлечения)
Сценарий: на ресепшене клиники владелец сканирует QR → ставит приложение → визиты/календарь питомца сразу доступны (deep-link + привязка owner_clinic_links).
Экран приветствия после deep-link: «Ваша клиника добавила питомца » + согласия.
3.3 Первый питомец
Пошаговый ProgressStepper; добавление питомца (фото, вид, кличка, дата рождения, порода — текст/справочник).
Empty-онбординг на «Главной», если питомцев нет: «+ Добавить питомца».


4. Раздел «Главная» (Dashboard)
Цель: мгновенный срез состояния питомца + целевые действия. Фон: мятный градиент. Шапка: лого PetHealthAI + аватар активного питомца.
4.1 Pet Switcher (header)
Аватар + кличка + порода активного питомца; тап — горизонтальный список питомцев + «+ Добавить».
Данные: usePets (active_pet_id в PetContext), аватар из Storage/кэша.
Поверхность: glass-полоса в шапке.
4.2 Карточка «Вес» (Weight Tracking) — solid
Заголовок + «Последний вес: X кг, N дней назад» + линейный график (Aug→Nov), точки по weight_history.
Цвет линии/заливки — accent. Пустой график → empty-«Добавьте первое взвешивание».
4.3 Status-карточки (сетка 2×N) — solid
Круглый IconChip + заголовок + значение; цвет статуса — ok/warn/danger:

Вакцинация — ближайшая next_due_date (record_vaccines): «Защищён» / «Требуется ревакцинация»; иконка-шприц.
Ближайшая запись (Upcoming Appointment) — из appointments (status confirmed/requested): причина, клиника, врач, дата/время + бейдж клиники; иконка-стетоскоп + календарь. (см. §10.2)
Обработка от паразитов — раздельно ecto/endo (record_parasite_treatments): «Обработан до …».
Лекарства (Medication Tracker) — активные курсы (record_prescriptions); иконка-таблетки.
Анализы (Lab Results) — последние результаты; иконка-пробирка.
Daily Activity — баллы/прогресс активности; иконка-бег.
4.4 Карточка «Рекомендации врача» — solid (новое, US#17)
Заголовок + саб «с последнего визита: <клиника>, <дата>» + 1–3 пункта + «Все рекомендации» → деталь визита.
Источник: medical_records.recommendations последнего визита. Пусто → карточка скрывается. (см. §10.3)
4.5 AI Insight — glass
Иконка-бот + короткий совет (≤150 симв.) + «Обсудить с ИИ» → AssistantStack (чат с этим контекстом).
Источник: ежедневный сигнал (та же таблица, что лента ассистента).
4.6 Геймификация (Progress) — glass
Прогресс «До следующей помощи приюту» + баланс «лапок» (иконка-лапка).
Данные: useLoyaltyPoints (user_points, чтение; начисление server-side).
4.7 Быстрые действия
«Добавить документ» → камера → пайплайн OCR → Review (§5.3).
«Спросить ИИ» → чат с поднятой клавиатурой.

Состояния экрана: skeleton-карточки при загрузке; empty (нет питомца) → онбординг; error на каждой карточке независимо.


5. Раздел «Здоровье» (Medical Hub)
Шапка раздела + переключатель «Список ↔ Календарь» (segmented, активный — accent).
5.1 Паспорт питомца — solid
Карточка: фото, кличка, порода, возраст; ниже — характеристики (чип/клеймо, вес, стерилизация, группа крови).
Плашка «Аллергии» — danger-предупреждение высоко, только при наличии (US#14).
Секции «Аллергии» и «Хронические» — CRUD; pet_allergies.severity пикер (Лёгкая/Средняя/Тяжёлая/—).
Действия: «Редактировать», «Поделиться QR».
5.2 Журнал здоровья (Treatment Log) — solid
Режим «Список»: timeline медсобытий; вкладки Обзор / Вакцины / Лекарства / Записи; 3 типа иконок (вакцинация-шприц / паразиты-капля / осмотр-лупа); чипсы-фильтры сверху.
Тап по вакцине/лекарству → деталь родительской записи (они — дети записи).
Режим «Календарь» (новое, US#16): под-переключатель Месяц / Неделя / Повестка; marked dates по типам с легендой (визит/вакцина/паразиты/лекарство/запись); agenda выбранного дня снизу. v2 — строки доз лекарств с чек-«принято» → баллы (useLoyaltyPoints). (см. §10.1)
5.3 OCR Review — solid
Превью фото (миниатюра) + предзаполненная ИИ форма; поля с низкой confidence подсвечены.
Select тип (Прививка/Паразиты/Осмотр), AutocompleteInput препарат (пресеты medicalPresets), DateTimePicker дата, вес.
«Подтвердить и сохранить» → RPC save_medical_record → анимация начисления «лапок» (server-side award) → возврат на Dashboard с обновлёнными виджетами.
5.4 Настройка напоминаний — solid
Авто-появление после сохранения прививки/обработки.
Toggle «Напомнить за X дней»; интервал паразитов (30/90/180); время уведомления.
5.5 Архив документов (Vault) — solid
Сетка/список превью (сканы/PDF); тап → к какой записи привязан.
5.6 Деталь записи (Record Detail) — solid
Поля родителя + дети (вакцины/лекарства/анализы).
Секция «Рекомендации» (новое, US#17) — список пунктов визита; (опц.) пункт → «Добавить напоминание»/«В календарь».
«Изменить» (правит поля родителя), «Удалить» (каскад).


6. Раздел «Ассистент» (AI & Knowledge)
Все экраны — в AssistantStack. AI — OpenRouter через Edge ai-proxy.
6.1 Hub
Поиск по базе знаний; плитки-промпты; категории (кинолог, породы, релокация).
6.2 Чат (BuddyAI) — glass-баблы бота + glass-инпут
Шапка: аватар бота, «Smart AI Medical Bot», статус online; контекст активного питомца.
Баблы: пользователь — accent-заливка (текст on-accent); бот — glass/surface со свечением glow-accent.
Встроенные мини-виджеты данных (приём из референса): карточка «Health Timeline» прямо в переписке (узлы past/future, время). Рендерить как Card внутри потока.
Инпут — glass pill + кнопка отправки accent; streaming-индикатор; вложения; quick replies.
История — conversations/messages (useConversation, одна беседа на питомца).
6.3 Релокация
Селектор страны; чек-лист документов; словарь команд с TTS.
6.4 Статья
Контент + блок «спросить ИИ» по статье.
6.5 Визуальный анализатор (Vision)
Рамка сканирования; резюме по срочности (ok/warn/danger); через ai-proxy (purpose=vision).

10. Новые фичи (детали)
10.1 Календарь/органайзер (US#16) — в §5.2
Агрегатор-хук поверх Supabase: medical_records.occurred_at + reminders.due_date + развёртка record_prescriptions по дням + appointments.
react-native-calendars (month/week/agenda, marked dates). v1 read-only; v2 — medication_intakes + чек «принято» → баллы.
Легенда типов: визит/вакцина/паразиты/лекарство/запись. UI-текст — i18next.
10.2 Запись к врачу (US#15) — экраны
Book Appointment: выбор партнёрской клиники → дата/время → причина → питомец → «Записаться» (status requested); per-clinic согласие на передачу данных конкретной клинике в момент создания.
My Appointments: табы Предстоящие/История; карточка + StatusBadge (requested/confirmed/cancelled/completed); действия отмена/повтор.
Регистрация экранов в AppNavigator. Сущность appointments (pet_id, clinic_id, requested_at, reason, status, source). Уведомление клиники — webhook/почта (clinic_integrations).
10.3 Рекомендации врача (US#17)
Источник: medical_records.recommendations. Вывод: карточка на «Главной» (§4.4) + секция в детали визита (§5.6). (опц.) пункт → напоминание/событие календаря.
