Визуальный фундамент — PetHealthAI redesign (§1–2 ТЗ)
Источник: дизайн-референсы (7 AI-мудбордов) + «Задание на дизайн». Референсы взяты как эстетика, не как буквальные строки/числа (текст в них сгенерён и местами битый). Стек: React Native / Expo — токены живут в JS-теме (theme-объект + контекст), не в Tailwind. Тёмная тема — через theme-контекст. ⚠️ Акцент меняется с #6B4EFF на пастельный — обновить Задание на дизайн, конвенции проекта, CLAUDE.md.


1. Принципы
Тёплый wellness-тон, мягкое стекло. Лёгкие пастельные градиенты, фростед-стекло на декоративных/оверлейных поверхностях, крупные скругления, рассеянные тени с цветным свечением.
Данные — плотные и контрастные. Медицинские данные (графики, цифры, медзаписи, статусы) — на сплошных карточках с высоким контрастом, НЕ на стекле. Стекло — там, где нет мелких данных.
Один акцент, консистентно. Единый бренд-акцент (см. §2.1) для CTA, активных состояний, иконок-чипов, выделений. Семантика здоровья (ok/скоро/просрочено) — отдельная триада, не путать с брендом.
Тактильность точечно. 3D-иллюстрации (бейджи, награды) — только в геймификации/charity, как акцент-моменты, не по всему UI.
Доступность поверх эстетики. Контраст текста ≥ WCAG AA в обеих темах; стекло никогда не несёт критический текст на «грязном» фоне без подложки.

2.7 Типографика
Рекомендуется дружелюбный округлый гротеск (как в референсах): варианты — Onest / Manrope / Nunito Sans. Подключение через expo-font.
Шкала: display (hero-числа, H1), h2, h3, body, caption. Веса: 700 (заголовки/метрики), 500 (акценты), 400 (тело).
Числовые метрики (вес, даты, баллы) — крупно и жирно, как «Weight Tracking / 32 kg» в референсе.
Лого: «PetHealthAI» — две краски, AI акцентом.
2.8 Иконография
Круглые тонированные icon-chips (accent-tint фон + accent-иконка): шприц (вакцинация), стетоскоп (визит/запись), таблетки (лекарства), пробирка (анализы), бегущий (активность) — как на дашборде-референсе.
Линейные иконки, средняя толщина, скруглённые концы.
3D-бейджи (золото/серебро/медь) — отдельный иллюстративный набор только для наград.

4. Тёмная тема
Обе темы равноправны. Реализация — theme-контекст (useTheme), все цвета только из токенов выше (никаких хардкод-hex в компонентах). Стекло в dark — затемнённое (rgba(40,40,52,0.5)), градиент-фон глубокий, акцент чуть ярче для контраста. Проверить контраст t1/t2 на surface и на стекле в обеих темах.
