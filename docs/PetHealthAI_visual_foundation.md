Визуальный фундамент — PetHealthAI redesign (§1–2 ТЗ)
Источник: дизайн-референсы (7 AI-мудбордов) + «Задание на дизайн». Референсы взяты как эстетика, не как буквальные строки/числа (текст в них сгенерён и местами битый). Стек: React Native / Expo — токены живут в JS-теме (theme-объект + контекст), не в Tailwind. Тёмная тема — через theme-контекст. ⚠️ Акцент меняется с #6B4EFF на пастельный — обновить Задание на дизайн, конвенции проекта, CLAUDE.md.


1. Принципы
Тёплый wellness-тон, мягкое стекло. Лёгкие пастельные градиенты, фростед-стекло на декоративных/оверлейных поверхностях, крупные скругления, рассеянные тени с цветным свечением.
Данные — плотные и контрастные. Медицинские данные (графики, цифры, медзаписи, статусы) — на сплошных карточках с высоким контрастом, НЕ на стекле. Стекло — там, где нет мелких данных.
Один акцент, консистентно. Единый бренд-акцент (см. §2.1) для CTA, активных состояний, иконок-чипов, выделений. Семантика здоровья (ok/скоро/просрочено) — отдельная триада, не путать с брендом.
Тактильность точечно. 3D-иллюстрации (бейджи, награды) — только в геймификации/charity, как акцент-моменты, не по всему UI.
Доступность поверх эстетики. Контраст текста ≥ WCAG AA в обеих темах; стекло никогда не несёт критический текст на «грязном» фоне без подложки.

2.2 Поверхности и стекло (дисциплина)
Обновлено в редизайне: канон ниже отражает реализацию (theme.js / Screen.js / GlassCard.js).

ФОН — radial-blob (обновлено в редизайне, заменяет прежний линейный bg-gradient).
Подложка bgBase: light #FBFEFD / dark #0F1117. Поверх — 3 мягких радиальных пастельных пятна (react-native-svg RadialGradient, реализовано в Screen.js); координаты cx/cy и радиус r — доли размера экрана (0..1), fade color → transparent по краю. Mint доминирует, peach/blue — деликатные.

Токен | Light | Dark | Где

border-hairline | rgba(0,0,0,0.06) | rgba(255,255,255,0.08) | разделители

saturate (data 1.4 / decor 1.9) — web-CSS свойство, в React Native НЕ применяется; читаемость компенсируется bg-альфой + intensity (= blur-токен).
RN-нюанс: backdrop-blur в RN — это expo-blur <BlurView> (не CSS backdrop-filter): BlurView (intensity = blur-токен, tint по схеме, на Android experimentalBlurMethod='dimezisBlurView') + полупрозрачный bg-оверлей токена + 1px бордер. Примитив GlassCard variant='data'|'decor'.

Правило: стекло — только из списка выше. Любая карточка с числами/графиком/списком записей = surface(solid) или data-стекло; decor — только декор/бренд.

2.2-bis Правила применения акцента (обновлено в редизайне — выведено на эталоне Dashboard)
accent — только точечно: иконки, активные состояния, прогресс-трек, бордеры, линия графика.
CTA-кнопки с белым текстом — заливка accentPress (НЕ accent): белый на accent #56B89F ≈ 2.4:1 (не проходит AA), на accentPress #3E9C84 ≈ 3.35:1 (крупный-AA). onAccent остаётся белым.
Фоны крупных поверхностей — surface/стекло, НЕ accent (избегать «переизбытка мятного»).
Числа/метрики — t1 (НЕ accent), иначе сливаются с фоном.
Семантика ok/warn/danger — только для реальных статусов здоровья (просрочка/норма/тревога), НЕ для нейтральных метрик (вес → t3).
Текст ≥ AA: тело/числа — t1/t2; accent-семейство (accentPress) — только для крупного CTA-текста, не для мелкого тела.

2.2-ter Категориальная палитра событий — eventTypes (обновлено в редизайне)
Третий вид цвета, рядом с accent (бренд) и семантикой (ok/warn/danger): для ТИПОВ событий в календаре/списке Medical, где нужна различимость нескольких категорий — это НЕ статус и НЕ бренд. НЕ зависит от accent-пресета (mint/peach/blue дают тот же набор). Per-scheme (light/dark; dark — чуть ярче для контраста на тёмном). Значения 1:1 с theme.eventTypes.

Тип | Light | Dark | Тон
record (визит/запись)    | #7B6EF0 | #9A8FF5 | violet
prescription (лекарство) | #4F8DF0 | #6BA1F5 | blue
vaccine (вакцина)        | #2BB6A8 | #4FCFC0 | teal (brand-adjacent)
reminder (напоминание)   | #E0A23E | #ECB75C | amber
appointment (приём)      | #E06C9C | #EC88B2 | rose

Правило: палитра намеренно избегает зелёного/красного — чтобы тип не читался как статус здоровья (ok/danger). Применяется к маркерам календаря, легенде и иконкам типов в agenda/списке — ОДИН цвет на тип во всех трёх местах (единообразие). Подписи легенды — i18n (calendar.types.*), текст t2.

2.7 Типографика
Рекомендуется дружелюбный округлый гротеск (как в референсах): варианты — Onest / Manrope / Nunito Sans. Подключение через expo-font.
Шкала: display (hero-числа, H1), h2, h3, body, caption. Веса: 700 (заголовки/метрики), 500 (акценты), 400 (тело).
Числовые метрики (вес, даты, баллы) — крупно и жирно, как «Weight Tracking / 32 kg» в референсе.
Лого: «PetHealthAI» — две краски, AI акцентом.
2.8 Иконография
Круглые тонированные icon-chips (accent-tint фон + accent-иконка): шприц (вакцинация), стетоскоп (визит/запись), таблетки (лекарства), пробирка (анализы), бегущий (активность) — как на дашборде-референсе.
Линейные иконки, средняя толщина, скруглённые концы.
3D-бейджи (золото/серебро/медь) — отдельный иллюстративный набор только для наград.

4. Тёмная тема
Обе темы равноправны. Реализация — theme-контекст (useTheme), все цвета только из токенов выше (никаких хардкод-hex в компонентах). Стекло в dark — затемнённое (rgba(40,40,52,0.5)), градиент-фон глубокий, акцент чуть ярче для контраста. Проверить контраст t1/t2 на surface и на стекле в обеих темах.
