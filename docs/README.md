# docs/ — проектная документация PetHealthAI

Дизайн-доки редизайна и связанные планы. Код и конвенции живут в коде и в
корневом `CLAUDE.md` (его Claude Code читает автоматически из корня репозитория);
здесь — развёрнутые спецификации и решения, на которые `CLAUDE.md` ссылается.

## Источники истины

> **Источник истины — версии в этом `docs/` (репозиторий); копии в базе знаний проекта — зеркало, при расхождении главная — репо.**

| Файл | Что это | Статус |
|------|---------|--------|
| [`PetHealthAI_visual_foundation.md`](./PetHealthAI_visual_foundation.md) | **Дизайн-токены** — палитра (мятный `accent`, `ok/warn/danger`), поверхности, дисциплина стекла (data/decor), радиусы, тени, типографика. | **Источник истины по визуальному фундаменту** (реализован в `src/theme/theme.js`). |
| [`PetHealthAI_design_spec_full.md`](./PetHealthAI_design_spec_full.md) | **Поэкранное ТЗ** — требования к каждому экрану (компоновка, состояния, поведение). | **Источник истины по экранам.** |
| [`PetHealthAI_redesign_kickoff.md`](./PetHealthAI_redesign_kickoff.md) | **Порядок работ и DoD** — фазы редизайна, последовательность, критерии готовности. | Процесс/план. |
| [`PetHealthAI_docs_update_plan.md`](./PetHealthAI_docs_update_plan.md) | План обновления документации по ходу редизайна. | Процесс/план. |
| [`PetHealthAI_CLAUDE_patch.md`](./PetHealthAI_CLAUDE_patch.md) | Правки конвенций для `CLAUDE.md` (что именно меняется в правилах проекта). | Изменения конвенций. |

## Как пользоваться

- **Меняешь токены/цвет/стекло** → сверяйся с `visual_foundation`; правки кода — в `src/theme/`.
- **Меняешь экран** → сверяйся с `design_spec_full`; держи реализацию в соответствии с ТЗ.
- **Планируешь фазу / проверяешь готовность** → `redesign_kickoff` (порядок + DoD).
- **Затрагиваешь ТЗ/интерфейс/схему/бизнес-модель** → отметь, какой документ обновить (см. раздел в корневом `CLAUDE.md`), не расходись с доками молча.

## Связь с кодом

- Токены реализованы в `src/theme/theme.js` + `src/theme/ThemeProvider.js` (Redesign Фаза 1a).
- Конвенции (theme-токены вместо хардкод-hex, мятный акцент, дисциплина стекла) — в корневом `CLAUDE.md`, раздел «Тема и визуальная система (редизайн)».
